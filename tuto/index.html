Remplir les infos du client

Ouvre le fichier config.json à la racine du projet et remplace chaque valeur.

Règle d'or : ne touche à rien d'autre que config.json et le dossier images/. Tout le reste se met à jour automatiquement.

Informations générales

"siteName"
Nom du garage — apparaît dans la barre du navigateur et le header.
"Garage Dupont Auto"
"slogan"
Accroche courte sous le nom.
"Votre expert automobile à Lyon"
"badge"
Petit texte affiché dans le badge du hero (haut de page).
"Mécanicien certifié · 15 ans d'expérience"
"description"
Paragraphe de présentation du garage. Utilisé dans la section À propos et le footer.
"Spécialistes en réparation et entretien automobile depuis 15 ans..."

Couleurs

"primary"
Couleur principale (boutons, titres colorés, accents). Format hexadécimal.
"#E74C3C" = rouge · "#1a73e8" = bleu
"secondary"
Couleur secondaire (dégradés, hero).
"#2C3E50"
Pour trouver un code couleur, va sur coolors.co ou tape "color picker" dans Google. Le code commence toujours par #.

Textes du hero (grande bannière d'accueil)

"title"
Titre principal en grand.
"Réparation auto rapide et transparente"
"subtitle"
Phrase d'accroche sous le titre.
"Devis gratuit en 24h, intervention sous 48h."
"cta_primary"
Texte du bouton principal.
"Prendre rendez-vous"
"cta_secondary"
Texte du bouton secondaire.
"Nos services"
"tags"
Petites étiquettes affichées sous les boutons du hero — arguments rapides (3 max).
["Toutes marques", "Sans rendez-vous", "Véhicule de prêt"]

Stats (section À propos)

"stats"
3 chiffres clés du garage. Chaque entrée a un number (le chiffre) et un label (la description).
"20+" / "Années d'expérience"

Services proposés

"services"
Liste des prestations (généralement 3). Chaque service a un title, une description, et un icon.
Pour l'icône, utilise n'importe quel code FontAwesome :
"fas fa-cogs" · "fas fa-wrench" · "fas fa-car" · "fas fa-oil-can"
Trouve une icône sur fontawesome.com/search — cherche en anglais, copie le code (ex. fas fa-car-crash).

Contact

"address"
Adresse complète.
"12 Rue de la Mécanique, 69200 Vénissieux"
"phone"
Numéro affiché et cliquable (appel direct sur mobile).
"04 72 51 83 92"
"whatsapp"
Numéro WhatsApp au format international sans + ni espaces (active le bouton flottant).
"33472518392" pour le 04 72 51 83 92
"email"
Email de contact, cliquable automatiquement.
"contact@garage-dupont.fr"
"hours"
Horaires affichés dans la section Contact.
"Lun–Ven : 8h–18h · Sam : 8h–12h"

Badge "Ouvert / Fermé" dans le hero

Le site calcule automatiquement si le garage est ouvert en ce moment et affiche un badge en haut de page. Renseigne les plages horaires :

"schedule": [ { "days": [1,2,3,4,5], "open": "08:00", "close": "18:00" }, // lun–ven { "days": [6], "open": "08:00", "close": "12:00" } // samedi ]
Les jours : 1 = lundi, 2 = mardi … 6 = samedi, 0 = dimanche. L'heure format 24h, toujours entre guillemets. Laisse "schedule": [] pour masquer le badge.

FAQ (questions fréquentes)

La section FAQ apparaît automatiquement si le tableau est rempli, disparaît si vide. Ajoute autant de questions que nécessaire :

"faq": [ { "question": "Acceptez-vous toutes les marques ?", "answer": "Oui, toutes marques et modèles." }, { "question": "Faut-il prendre rendez-vous ?", "answer": "Non, nous acceptons sans rendez-vous." } ]

Réseaux sociaux

URL complète de la page Facebook. Laisse "" si pas de page.
"https://facebook.com/garagedupont"
"instagram"
URL du profil Instagram.
"https://instagram.com/garagedupont"
2

Ajouter les photos du garage

Dépose les images dans le dossier images/ avec les bons noms.

Système de slots : chaque image a un nom réservé. Il suffit de déposer la photo avec le bon nom dans images/ — elle apparaît automatiquement au bon endroit.
hero.jpg
Photo de fond de la grande bannière d'accueil. Idéalement 1920×1080px. Si absent → dégradé coloré.
about.jpg
Photo de la section "À propos" (équipe, atelier…). Si absent → icône décorative.
gallery-1.jpg à gallery-6.jpg
Photos de la galerie (véhicules réparés, atelier, équipe…). 6 photos max. Format 800×600px conseillé.
Les noms sont sensibles à la casse — écris-les exactement comme indiqué (gallery-1.jpg et non Gallery-1.JPG). Les formats acceptés : .jpg, .jpeg, .png, .webp.

Mets ensuite les noms exacts dans le config.json sous la clé "gallery" :

// Dans config.json → "images" "images": { "hero": "hero.jpg", "about": "atelier.jpg", "gallery": [ "gallery-1.jpg", "gallery-2.jpg", "gallery-3.jpg" ] }
3

Intégrer Google Maps

Une carte interactive apparaîtra dans la section Contact.

1

Va sur maps.google.com et recherche l'adresse exacte du garage.

2

Clique sur Partager (icône de partage), puis sur l'onglet "Intégrer une carte".

3

Clique sur "Copier le code HTML". Tu obtiens quelque chose comme :
<iframe src="https://www.google.com/maps/embed?pb=...">

4

Copie uniquement l'URL qui est dans le src="..." et colle-la dans config.json :

"maps_embed_url": "https://www.google.com/maps/embed?pb=!1m18..."
Si tu laisses "maps_embed_url": "", la carte ne s'affiche tout simplement pas — aucun problème.
4

Renseigner les avis Google

Affiche le badge de note et les vrais avis des clients.

Badge de note globale

Renseigne la note, le nombre d'avis et le lien vers la fiche Google du garage :

"google_reviews": { "rating": 4.9, // note sur 5 "count": 87, // nombre total d'avis "url": "https://g.page/r/..." // lien fiche Google }
Pour trouver le lien : va sur Google Maps → fiche du garage → Partager → copie le lien court (https://g.page/r/...). Si la fiche n'a pas de lien court, laisse "".

Ajouter des témoignages

Copie les vrais avis Google dans la section "testimonials". Ajoute "source": "google" pour afficher le logo Google à côté.

{ "text": "Super garage, travail rapide et prix honnête !", "author": "Jean-Pierre M.", "role": "Client", "rating": 5, "source": "google" }
5

Activer le formulaire de contact

Les messages arrivent par email via EmailJS — zéro spam, zéro redirection.

Le formulaire utilise EmailJS (gratuit jusqu'à 200 emails/mois). C'est toi le développeur qui crées le compte une fois, le client n'a rien à faire.
1

Crée un compte sur emailjs.com (gratuit).

2

Ajoute un Email Service — connecte l'adresse email du client (Gmail, SMTP…). Note le Service ID.

3

Crée un Email Template avec les variables {{from_name}}, {{from_email}}, {{phone}}, {{message}}. Dans "To Email", mets l'adresse du client. Note le Template ID.

4

Dans Account → API Keys, copie ta Public Key.

5

Colle les 3 valeurs dans config.json :

"emailjs": { "service_id": "service_xxxxxxx", "template_id": "template_xxxxxxx", "public_key": "xxxxxxxxxxxx" }
Si la section "emailjs" est absente ou vide, le formulaire reste visible et affiche une notification locale — parfait pour les démos client.
6

Mettre en ligne

Le site se redéploie automatiquement à chaque modification poussée.

1

Ouvre le terminal dans le dossier du projet.

2

Tape ces 3 commandes :

git add . git commit -m "Nouveau client — Garage Dupont" git push
3

Attends 30 secondes — Vercel redéploie automatiquement. Le site est en ligne.

Avant de livrer au client, teste sur mobile — la majorité des visiteurs d'un garage naviguent sur téléphone. Vérifie que le bouton WhatsApp fonctionne bien.
